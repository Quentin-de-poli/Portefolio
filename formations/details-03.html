How to program in Godot - GDScript Tutorial

Formation : Godot University | Dossier source : image_d_03

Introduction :

Présentation globale de GDScript, le language de base de programmation de Godot (conçu spécialement pour lui).

Hello World ! :

Le classique de tout tutoriel de programation, on apprend à afficher un message

La syntaxe :

On apprend la sensibilité du code face a l'indentation, les majuscules, l'abscence de point-virgule, etc...

Modifying nodes 1.0

Apprendre à modifier des nodes en récupérant leur code, on apprnedra a modifier le texte, sa couleur, etc...

Les Input :

On configure l'intput map, ce qui sert a assigner des touches à des actions (ex : up = "z", down = "s"). Ensuite on code un programme pour les utiliser

Variables 1.0 :

Introduction aux variables, leur déclaration et leur utilisation (ici des maths basiques)

If statements :

Dans cette partie, on nous apprend a utiliser les "if", "elif" et "else" pour créer des conditions dans notre code.

Comments :

Cette partie semble simple mais essentielle pour rendre le code plus lisible et compréhensible. Grace aux commentaires, on peut ecrire du texte qui ne sera pas exécuté par le programme.

Variables 2.0

Les variables ont plusieurs types, c'est ce qu'on apprend ici, les integer, les float, les string, etc. On apprend aussi qu'on peut les convertir au cours de l'exécution du programme.

Functions :

Essentiel pour structurer son code et eviter les répétitions, les fonctions permettent de regrouper du code réutilisable.

Random numbers :

Utilisation de fonctions comme randf et randi_range pour generer des valeurs aléatoires.

Documentation :

Présentation de la documentation, cliquer sur un element du code pour voir son utilité

Arrays :

On a créé/manipulé des listes permettant de stocker plusieurs valeurs dans une seule variable.

Loops :

On apprend a utiliser les boucles for et while pour repeter des action (comme parcourir une liste)

Dictionaries :

Organisation de données sous forme de paires clé-valeur.(exemple : clé = nom d'un joueur, valeur = niveau)

Enums :

On a créé des listes de status personnalisé(ex : allié/ennemi) ce qui est utile pour faciliter les différent états que peut prendre un objet ou pnj

Match :

ça permet d'executer différente actions en fonction de la valeur d'une variable. Tres pratique pour les "enums"

Modifying nodes 2.0 :

On complexifie la modification des nœuds en utilisant de nouvelles méthodes, comme onready, le "$", get_path, et plein d'autres encore.

Signals :

Système de communication entre les nœuds, permettant de déclencher des actions en réponse à des événements spécifiques.

Get/Set :

Ces petit accessoires sont bien pratique, ils permettent de lire et d'écrire les propriétés des nœuds de manière simplifiée.

Classes :

Les classes permettent de créer des objets personnalisés avec leurs propres propriétés et méthodes. Exemple : plusieurs classes de pnj (forgeron, vendeur de potions)

Inner classes :

Se sont des petites classes définies à l'intérieur d'un script, ça permet de grouper des données en toutes sécurités.

Inheritance :

C'est l'heritage entres des Noeuds, comme par exemple camera2D fait herite de Node2D.

Composition :

C'est une alternative a L'Inheritance et elle serait meilleure pour structurer le code.

Call down, signal up :

La on étudie la relation parent-enfant dans le contexte des signaux. (ex : un noeud enfant peut utiliser des fonctions qu'il hérite du nœud parent, mais pas l'inverse).
